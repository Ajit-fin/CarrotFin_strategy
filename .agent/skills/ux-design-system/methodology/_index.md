# UX Design System — Methodology Inventory

> **Last updated:** 2025-04-15

---

## Available Checklists

### Adaptive Behavior Matrix Checklist

When designing any component, verify it defines behavior across ALL 6 user state dimensions:

- [ ] **Financial literacy** (1-5) — how does content depth, jargon, and explanation change?
- [ ] **Risk tolerance** (conservative → aggressive) — how does framing and recommendation confidence change?
- [ ] **Life stage** (single → family → pre-retirement) — how does priority and relevance change?
- [ ] **Engagement mode** (conversational → visual → hybrid) — how does interaction modality change?
- [ ] **Emotional state** (calm → anxious → excited) — how does tone and pacing change?
- [ ] **Trust level** (new → warming → trusting → dependent) — how does data request depth and recommendation assertiveness change?

If a dimension genuinely doesn't apply to this component, document WHY, not just skip it.

---

### Composition Audit Checklist

When assembling a generative screen composition, verify:

- [ ] Max 5-7 components per surface
- [ ] Highest-priority item at top (priority order documented)
- [ ] Single primary CTA visible
- [ ] Anchor elements (action button, nav, AI persona) maintain spatial position
- [ ] Progressive disclosure — headline first, details on interaction
- [ ] At least one actionable element (not purely informational)
- [ ] Cognitive load appropriate for target literacy level
- [ ] Screen type matches decision framework from `screen-taxonomy.md`

---

### Conversational Integration Checklist

When designing any visual component, verify:

- [ ] AI introduction trigger defined — what causes the AI to place this component?
- [ ] Conversational entry — how does the AI verbally introduce or explain this component?
- [ ] User interaction response — what happens when the user interacts? Does the AI respond conversationally?
- [ ] Handoff pattern — if the user needs more detail, does the conversation expand inline or pivot to a new surface?
- [ ] Ambient layer connection — can this component appear as a nudge? If so, what triggers the nudge?

---

*Add new checklists as design methodology evolves. Checklists should be actionable, not aspirational.*
